## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 한국 중소/중견 기업은 글로벌 서비스데스크 솔루션의 불완전한 한국어 지원, 카카오톡 알림 부재, 계층형 승인 구조 미지원으로 서비스 운영에 마찰을 겪고 있다. |

## 1. Architecture Overview

### 1.1 선택 아키텍처: Option C — Pragmatic Balance

비즈니스 로직이 API Routes에 직접 구현되거나 lib/ 유틸리티를 통해 처리되는 실용적 구조이다.
별도의 services/ 및 repositories/ 레이어 없이 Next.js App Router의 API Routes가 thin controller 역할을 하면서 동시에 비즈니스 로직을 직접 처리한다.

- **API Routes** (controller + business logic): 요청 검증, 도메인 로직, 상태 전이 조정, 트랜잭션 관리
- **lib/** (shared utilities): State Machine, Business Hours Engine, 세션, 채번 등 공유 유틸리티
- **jobs/** (batch scheduler): BullMQ 기반 7개 배치 잡

### 1.2 핵심 설계 결정

| 결정 사항 | 선택 | 근거 |
|-----------|------|------|
| 상태 관리 | State Machine 패턴 (centralized) | 9개 상태, 17개 전이 이벤트 — 중앙 집중식 관리로 일관성 보장 |
| 배치 스케줄러 | BullMQ + Redis | 지속 큐, 재시도, 스케줄링, 순차 실행 지원 |
| 세션 관리 | Server Session + Redis | HttpOnly 쿠키 + Sliding Expiry 8시간 |
| 파일 저장소 | Cloudflare R2 (Presigned URL) | S3 호환 API, 비용 효율적 |
| Push 알림 | Web Push (VAPID) | Phase 1 즉시 제공 가능, Service Worker 기반 |

## 3. Database Schema

### 3.1 Prisma Schema 전체 정의

```prisma
// prisma/schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

// === onDelete 전략 ===
// Restrict: 참조 데이터 존재 시 삭제 차단 (Company, Project, Ticket 등 핵심 엔티티)
// SetNull: 참조 대상 삭제 시 null 설정 (선택적 관계)
// 비활성화는 소프트 딜리트(isActive=false)로 처리, hard delete는 원칙적으로 차단

// ============================================================
// 1. users — 통합 사용자 테이블 (FR-03)
// ============================================================
model User {
  id                  String    @id @default(cuid())
  loginId             String    @unique @map("login_id")
  password            String                                // bcrypt 해시
  name                String
  email               String?   @unique                    // 선택 (V1.1: optional로 변경)
  phone               String?                              // 선택 (V1.1: optional로 변경)
  type                UserType                               // admin / support / customer
  isActive            Boolean   @default(true) @map("is_active")
  mustChangePassword  Boolean   @default(true) @map("must_change_password")
  loginAttempts       Int       @default(0) @map("login_attempts")
  lockedUntil         DateTime? @map("locked_until")
  companyId           String?   @map("company_id")           // customer만 필수
  departmentId        String?   @map("department_id")        // customer 선택
  createdAt           DateTime  @default(now()) @map("created_at")
  updatedAt           DateTime  @updatedAt @map("updated_at")

  // Relations
  company             Company?            @relation(fields: [companyId], references: [id], onDelete: SetNull)
  department          Department?         @relation(fields: [departmentId], references: [id], onDelete: SetNull)
  projectMembers      ProjectMember[]
  ticketsRegistered   Ticket[]            @relation("RegisteredBy")
  ticketsCustomer     Ticket[]            @relation("CustomerUser")
  ticketAssignments   TicketAssignment[]
  comments            Comment[]
  notifications       Notification[]
  pushSubscriptions   PushSubscription[]
```

```typescript
interface ApiResponse<T> {
```

### 5.3 전이 테이블 (17개 이벤트)

| From State | Event | To State | Guard Conditions | Side Effects |
|-----------|-------|----------|-----------------|--------------|
| REGISTERED | `MANUAL_RECEIVE` | IN_PROGRESS | 프로젝트 support 멤버, 낙관적 락 | deadline=expectedCompletionDate, receivedAt=now(), 이력, 알림 |
| REGISTERED | `AUTO_RECEIVE` | RECEIVED | 4근무시간 경과 (배치) | deadline=desiredDate, 이력(actor=SYSTEM), 알림 |
| REGISTERED | `SUPPORT_REGISTER` | IN_PROGRESS | 지원담당자 직접 등록 | receivedAt=createdAt, 동일 트랜잭션 즉시 IN_PROGRESS |
| REGISTERED | `CANCEL` | CANCELLED | 등록자 본인 또는 admin | 상태이력 기록 |
| RECEIVED | `CONFIRM_NORMAL` | IN_PROGRESS | now() <= deadline | 상태이력, 고객 알림 |
| RECEIVED | `CONFIRM_DELAYED` | DELAYED | now() > deadline | 상태이력, 고객+Main담당 알림 |
| IN_PROGRESS | `DEADLINE_EXCEEDED` | DELAYED | now() > deadline (배치) | 이력(actor=SYSTEM), 알림 |
| IN_PROGRESS | `REQUEST_EXTEND` | EXTEND_REQUESTED | businessHoursUntilDeadline >= 8, 이전 연기요청 없음 | ExtendRequest 생성, 고객 알림 |
| IN_PROGRESS | `REQUEST_COMPLETE` | COMPLETE_REQUESTED | - | CompleteRequest 생성 (attempt++), 고객 알림 |
| DELAYED | `REQUEST_COMPLETE` | COMPLETE_REQUESTED | - | CompleteRequest 생성 (attempt++), 고객 알림 |
| EXTEND_REQUESTED | `APPROVE_EXTEND` | IN_PROGRESS | 고객담당자 또는 자동승인 (4근무시간) | deadline=newDeadline, completeRequestCount=0, 이력, 알림 |
| EXTEND_REQUESTED | `REJECT_EXTEND` | IN_PROGRESS | 고객담당자, 반려 사유 필수 | 반려 후 즉시 기한 재확인 → now()>deadline이면 DELAYED |
| COMPLETE_REQUESTED | `APPROVE_COMPLETE` | SATISFACTION_PENDING | 고객담당자 | CompleteRequest.status=APPROVED, 알림 |
| COMPLETE_REQUESTED | `REJECT_COMPLETE` | (이전 상태) | 고객담당자, attempt <= 2 | 2회차 반려 시 관리책임자 알림. 복귀 후 기한 재확인 |
| COMPLETE_REQUESTED | `AUTO_APPROVE_3RD` | SATISFACTION_PENDING | attempt === 3 (자동) | autoApproved=true, 고객 알림 |
| SATISFACTION_PENDING | `RATE` | CLOSED | 고객담당자, 별점 1~5 | SatisfactionRating 생성 |
| SATISFACTION_PENDING | `AUTO_CLOSE` | CLOSED | 5근무일 경과 (배치) | 이력(actor=SYSTEM) |

### 6.3 Job 1: auto-receive

**주기:** 1분 | **대상:** FR-15 | **감지:** REGISTERED 상태 4근무시간 경과 → RECEIVED 전환

```typescript
// jobs/auto-receive.job.ts
async function autoReceiveJob() {
  const now = new Date();
  const registeredTickets = await prisma.ticket.findMany({ where: { status: 'REGISTERED' } });

  for (const ticket of registeredTickets) {
    const elapsed = businessHoursEngine.calculateBusinessHours(ticket.createdAt, now);
    if (elapsed >= 4) {
      const receivedAt = businessHoursEngine.addBusinessHours(ticket.createdAt, 4);
      await prisma.$transaction(async (tx) => {
        await tx.ticket.update({
          where: { id: ticket.id, status: 'REGISTERED' }, // 낙관적 락
          data: { status: 'RECEIVED', expectedCompletionDate: ticket.desiredDate,
                  deadline: ticket.desiredDate, receivedAt },
        });
        await tx.ticketStatusHistory.create({
          data: { ticketId: ticket.id, previousStatus: 'REGISTERED',
                  newStatus: 'RECEIVED', actorType: 'SYSTEM' },
        });
        await tx.ticketDeadlineHistory.create({
          data: { ticketId: ticket.id, previousDeadline: null,
                  newDeadline: ticket.desiredDate, reason: '자동접수' },
        });
      });
      await notificationService.notifyTicketReceived(ticket);
    }
  }
}
```

### 6.4 Job 2: delay-detect

**주기:** auto-receive 완료 후 체인 | **대상:** FR-16 | **감지:** IN_PROGRESS/RECEIVED 기한 초과 → DELAYED

```typescript
// jobs/delay-detect.job.ts
async function delayDetectJob() {
  const now = new Date();
  const overdueTickets = await prisma.ticket.findMany({
    where: { status: { in: ['IN_PROGRESS', 'RECEIVED'] }, deadline: { lt: now } },
  });
  for (const ticket of overdueTickets) {
    await prisma.$transaction(async (tx) => {
      await tx.ticket.update({ where: { id: ticket.id, status: ticket.status }, data: { status: 'DELAYED' } });
      await tx.ticketStatusHistory.create({
        data: { ticketId: ticket.id, previousStatus: ticket.status,
                newStatus: 'DELAYED', actorType: 'SYSTEM', reason: '처리기한 초과 자동 전환' },
      });
    });
    await notificationService.notifyDelayedTransition(ticket);
  }
}
```

```typescript
export function middleware(request: NextRequest) {
  const sid = request.cookies.get('sid');
  if (!sid && !request.nextUrl.pathname.startsWith('/login')) {
    return NextResponse.redirect(new URL('/login', request.url));
  }
  if (ADMIN_PATHS.some(p => request.nextUrl.pathname.startsWith(p))) {
    const roleHint = request.cookies.get('role_hint')?.value;
    if (roleHint !== 'admin') {
      return NextResponse.redirect(new URL('/dashboard', request.url));
    }
  }
  return NextResponse.next();
}
```

```typescript
// DB 저장 + Push 발송 (Push 실패 시에도 DB 알림은 보존)
async function sendNotification(params: {
  userId: string; ticketId?: string; type: NotificationType;
  title: string; body: string; linkUrl?: string;
}): Promise<void> {
  await prisma.notification.create({ data: params });

  const subscriptions = await prisma.pushSubscription.findMany({
    where: { userId: params.userId }
  });

  for (const sub of subscriptions) {
    try {
      await webpush.sendNotification(sub, JSON.stringify({
        title: params.title, body: params.body, url: params.linkUrl,
        tag: `${params.type}-${params.ticketId}`,
      }));
    } catch (error) {
      if ((error as any).statusCode === 410) {
        await prisma.pushSubscription.delete({ where: { id: sub.id } });
      }
    }
  }
}
```

## 12. Business Hours Engine Design

### 12.1 SPOF 식별

근무시간 계산 엔진은 시스템 전체의 **Single Point of Failure**. 자동접수(4근무시간), 연기요청 가능 여부(8근무시간), 연기 자동승인(4근무시간), 만족도 자동종료(5근무일), 지연감지 등 모든 시간 기반 로직이 이 모듈에 의존한다.

**V1.1 변경:** 엔진 캐시 TTL = `ENGINE_CACHE_MS = 5 * 60 * 1000` (5분)

### 12.2 인터페이스 정의

### 12.3 구현 핵심 로직

```typescript
// 설정값
const DEFAULT_WORK_START = '09:00'; // Asia/Seoul
const DEFAULT_WORK_END = '18:00';
const TIMEZONE = 'Asia/Seoul';
const ENGINE_CACHE_MS = 5 * 60 * 1000; // V1.1: 5분 캐시

class BusinessHoursEngineImpl implements BusinessHoursEngine {
  private holidays: Set<string>; // 'YYYY-MM-DD' format
  private workStart: { hour: number; minute: number };
  private workEnd: { hour: number; minute: number };

  calculateBusinessHours(from: Date, to: Date): number {
    let cursor = toSeoulTime(from);
    const end = toSeoulTime(to);
    let totalMinutes = 0;

    while (cursor < end) {
      if (this.isWorkingDay(cursor)) {
        const effectiveStart = max(cursor, setTime(cursor, this.workStart));
        const effectiveEnd = min(end, setTime(cursor, this.workEnd));
        if (effectiveStart < effectiveEnd) {
          totalMinutes += differenceInMinutes(effectiveEnd, effectiveStart);
        }
      }
      cursor = startOfNextDay(cursor);
    }
    return totalMinutes / 60;
  }
}
```

### 12.4 공휴일 DB 연동 + 캐싱

```typescript
async function createBusinessHoursEngine(): Promise<BusinessHoursEngine> {

  const holidays = await prisma.holiday.findMany({
    where: { year: { in: [currentYear, currentYear + 1] } },

  const workEnd = settings.find(s => s.key === 'work_end_time')?.value ?? '18:00';
  return new BusinessHoursEngineImpl(holidays.map(h => h.date), workStart, workEnd);
}
```

- 연속 공휴일 (설/추석) 경계
- 근무 시작/종료 경계값 (정확히 09:00, 18:00)
- 연말연시 (12/31 → 1/1 공휴일)
- 대체공휴일 처리

## 용어 정의 (Quick Reference)

| 용어 | 정의 |
|------|------|
| **처리희망일** | 고객담당자가 티켓 등록 시 입력하는 희망 처리 날짜 (기본값: 등록일 +5근무일) |
| **완료예정일** | 지원담당자가 수동 접수 시 입력하는 처리 완료 목표 날짜 |
| **연기요청일** | 연기요청 승인 시 확정된 새 완료 날짜 |
| **처리기한** | 티켓의 현재 유효한 기한 — 상태에 따라 자동 변경 |
| **자동접수** | 4근무시간 경과 시 시스템이 자동으로 접수 처리 |
| **Main 담당자** | 프로젝트의 주 지원담당자 (1명 필수) |
| **근무일/근무시간** | 평일 09:00~18:00 기준 (설정 가능), 공휴일 제외 |
| **관리책임자** | 시스템 설정에서 지정된 admin 계정 (완료요청 2회차 반려 시 알림 수신) |
