## Executive Summary

| 관점 | 설명 |
|------|------|
| **Problem** | 한국 기업은 글로벌 서비스데스크 솔루션(Zendesk, Freshdesk)의 불완전한 한국어 지원, 카카오톡/SMS 알림 부재, 한국형 계층적 승인 구조 미지원, 개인정보보호법 비준수 등으로 인해 서비스 운영에 마찰을 겪고 있다. |
| **Solution** | Option C (Pragmatic Balance) 아키텍처로 고객사-프로젝트-티켓 연계 구조와 한국형 다단계 승인 워크플로우, 자동접수 및 처리기한 자동 관리 기능을 갖춘 SaaS 티켓 기반 서비스데스크를 구축했다. 완전한 한국어 UI, Web Push 알림, 계층형 승인/보고 워크플로우, 국내 데이터 레지던시를 기본 제공한다. |
| **Functional UX Effect** | 고객담당자는 배정 프로젝트에서 즉시 티켓 등록 및 실시간 상태 확인이 가능하며, 지원담당자는 통합 티켓 중심 UI에서 접수-처리-완료 흐름을 효율적으로 수행한다. 관리자는 역할 기반 대시보드 및 온보딩 마법사로 팀 운영을 체계화할 수 있다. 서비스 요청자는 Web Push로 실시간 상태 알림을 받고, 담당자는 한국어 SLA 기반 대시보드로 응답 시간을 50% 단축하며, 관리자는 계층형 승인 워크플로우로 보고 체계를 일원화한다. |
| **Core Value** | 글로벌 솔루션 대비 30-50% 낮은 비용으로 한국 규정 완전 준수와 한국 비즈니스 문화에 최적화된 서비스데스크 경험을 제공하며, 고객사·프로젝트·담당자 연계 구조와 9상태·17이벤트 상태 머신, 자동접수, 계층형 승인 및 만족도 평가로 서비스 운영 신뢰도를 정량적으로 추적·관리한다. Beta 50팀 온보딩 준비 완료. |

## 2. PM Strategy: Value Proposition & Business Model

### 2.1 JTBD 6-Part Value Proposition

#### Jobs-To-Be-Done Statement

> **[상황]** IT 지원 요청이 이메일, 전화, 메신저로 분산되어 관리되지 않을 때,
> **[동기]** 모든 서비스 요청을 하나의 시스템에서 추적하고 SLA를 준수하고 싶다.
> **[기대 결과]** 서비스 품질이 투명하게 측정되고, 요청자 만족도가 향상되며, 팀의 업무 부하가 균등하게 분배된다.

#### 6-Part VP Canvas

| Part | 내용 |
|------|------|
| **1. Target Customer** | 한국 중소/중견 기업(50-500명)의 IT 운영팀, 고객지원팀, 총무/시설관리팀 |
| **2. Problem** | 글로벌 서비스데스크는 한국어 지원 불완전, 카카오톡 미연동, 한국형 승인 구조 미지원, 비싼 가격, 해외 서버 데이터 주권 문제가 있다 |
| **3. Promise** | 한국 기업에 최적화된, 합리적 가격의, 규정을 준수하는 서비스데스크를 제공한다 |
| **4. Proof** | Beta 고객 SLA 준수율 95%+, 카카오톡 알림 응답률 이메일 대비 2x, PIPC 준수 인증 |
| **5. Power (Moat)** | 한국어 네이티브 UX + 카카오톡 생태계 연동 + 국내 데이터 레지던시 + 한국형 워크플로우 = 글로벌 솔루션이 쉽게 복제할 수 없는 조합 |
| **6. Profit Model** | SaaS 구독 (agent당 월과금) + 프리미엄 티어 (자동화/AI) + Enterprise (온프레미스/전용 환경) |

## 부록 A: 티켓 상태 머신 & 비즈니스 규칙

### A.1 티켓 상태 (9개)

| 상태 코드 | 상태명 | 설명 |
|----------|--------|------|
| `REGISTERED` | 등록됨 | 티켓 최초 등록 상태 |
| `RECEIVED` | 접수됨 | 자동접수(4근무시간) 또는 수동접수 완료 |
| `IN_PROGRESS` | 처리중 | 담당자가 처리를 시작한 상태 |
| `DELAYED` | 지연중 | 처리기한 초과 (복귀 불가 — 패널티 성격) |
| `EXTEND_REQUESTED` | 연기요청중 | 연기요청 승인 대기 |
| `COMPLETE_REQUESTED` | 완료요청중 | 완료요청 승인 대기 |
| `SATISFACTION_PENDING` | 만족도대기 | 만족도 평가 대기 (5근무일 자동종료) |
| `CLOSED` | 종결 | 최종 종결 상태 |
| `CANCELLED` | 취소 | 티켓 취소 처리 |

### A.2 이벤트 목록 (17개)

| 이벤트 | 설명 |
|--------|------|
| `SUBMIT` | 티켓 등록 |
| `AUTO_RECEIVE` | 자동 접수 (4근무시간 경과, 배치) |
